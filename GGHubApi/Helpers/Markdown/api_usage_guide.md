# CS2 Tournament API - Інструкція використання
`namecheap - your-domain.com
 hostinger - your-domain2.com`

## Алгоритм створення та проведення турніру

### 1. Створення турніру

**Endpoint:** `POST /api/tournament`

**Запит:**
```json
{
  "title": "CS2 Championship",
  "description": "Professional CS2 tournament",
  "playersPerTeam": 5,
  "maxTeams": 8,
  "entryFee": 50.00,
  "maps": ["de_dust2", "de_mirage", "de_inferno"],
  "startTime": "2024-01-15T19:00:00Z",
  "rules": "Standard competitive rules"
}
```

**Параметри:**
- `createdBy` (query) - ID користувача-організатора

**Відповідь:**
```json
{
  "success": true,
  "data": {
    "id": "uuid",
    "title": "CS2 Championship",
    "status": "Created",
    "prizeFund": 360.00,
    "commission": 40.00,
    "totalRounds": 3,
    "paymentDeadline": "2024-01-15T17:00:00Z",
    "inviteLink": null
  }
}
```

### 2. Редагування турніру (до початку)

**Endpoint:** `PUT /api/tournament/{id}`

**Запит:**
```json
{
  "title": "Updated Championship",
  "entryFee": 60.00,
  "maxTeams": 16
}
```

### 3. Генерація інвайт-лінку

**Endpoint:** `POST /api/tournament/{id}/invite-link`

Інвайт-лінк формується на основі значення конфігурації `App:PublicBaseUrl`.

**Відповідь:**
```json
{
  "success": true,
  "data": "https://example.com/tournament/join/ABC123XY"
}
```

### 4. Приєднання команд до турніру

#### Варіант A: Створення нової команди
**Endpoint:** `POST /api/tournament/{id}/join`

**Запит:**
```json
{
  "teamName": "Team Liquid",
  "playerIds": ["uuid1", "uuid2", "uuid3", "uuid4", "uuid5"]
}
```

#### Варіант B: Приєднання за токеном (для членів команди)
**Endpoint:** `POST /api/tournament/join-by-token`

**Запит:**
```json
{
  "joinToken": "ABC123DEFG456",
  "userId": "uuid"
}
```

### 5. Оплата участі

**Endpoint:** `POST /api/tournament/payment`

**Запит:**
```json
{
  "tournamentId": "uuid",
  "teamId": "uuid",
  "amount": 50.00,
  "paymentProvider": "Cryptomus"
}
```

**Відповідь:**
```json
{
  "success": true,
  "data": {
    "id": "uuid",
    "paymentUrl": "https://pay.cryptomus.com/pay/abc123",
    "status": "Pending"
  }
}
```

### 6. Отримання турнірної сітки

**Endpoint:** `GET /api/tournament/{id}/bracket`

**Відповідь:**
```json
{
  "success": true,
  "data": {
    "tournamentId": "uuid",
    "totalRounds": 3,
    "currentRound": 1,
    "rounds": [
      {
        "round": 1,
        "roundName": "Quarter-Final",
        "matches": [
          {
            "id": "uuid",
            "team1": {"name": "Team A"},
            "team2": {"name": "Team B"},
            "status": "Ready",
            "serverIp": "192.168.1.100",
            "serverPort": 27015,
            "serverPassword": "match123"
          }
        ]
      }
    ]
  }
}
```

### 7. Запуск турніру

**Endpoint:** `POST /api/tournament/{id}/start`

**Умови:**
- Всі команди повинні оплатити участь
- Кількість команд = maxTeams
- Статус турніру = "PaymentCompleted"

### 8. Отримання інформації про матч

**Endpoint:** `GET /api/tournament/{id}/full`

**Відповідь містить:**
- Повну інформацію про турнір
- Список команд з гравцями
- Історію матчів
- Поточний статус

## SignalR Events для Real-time оновлень

### Підключення до Hub
```javascript
const connection = new signalR.HubConnectionBuilder()
  .withUrl("/tournament-hub")
  .build();

// Приєднатися до групи турніру
connection.invoke("JoinTournamentGroup", tournamentId);
```

### Події турніру
- `TournamentUpdated` - турнір оновлено
- `TeamJoined` - команда приєдналася
- `TeamLeft` - команда вийшла
- `PaymentCompleted` - оплата завершена
- `TournamentStarted` - турнір почався
- `MatchStarted` - матч почався
- `MatchCompleted` - матч завершено
- `ServerReady` - сервер готовий
- `TournamentCompleted` - турнір завершено

### Приклад обробки подій
```javascript
connection.on("TeamJoined", (team) => {
  console.log("New team joined:", team);
  updateTeamsList(team);
});

connection.on("ServerReady", (serverInfo) => {
  console.log("Server ready:", serverInfo);
  showConnectButton(serverInfo);
});
```

## Webhook обробка

### DatHost Webhook
**Endpoint:** `POST /api/webhook/dathost`

**Payload:**
```json
{
  "event": "match_end",
  "matchId": "external-match-id",
  "status": "completed",
  "winner": "team_a",
  "customData": "tournament-match-uuid",
  "score": {
    "team1": 16,
    "team2": 14
  }
}
```

### Cryptomus Webhook
**Endpoint:** `POST /api/webhook/cryptomus`

**Headers:**
- `sign`: MD5 підпис

**Payload:**
```json
{
  "uuid": "payment-uuid",
  "status": "paid",
  "orderId": "tournament-uuid-team-uuid",
  "amount": 50.00,
  "currency": "USD"
}
```

## Статуси та переходи

### Статуси турніру
1. `Created` → `WaitingForTeams`
2. `WaitingForTeams` → `WaitingForPayments`
3. `WaitingForPayments` → `PaymentCompleted`
4. `PaymentCompleted` → `InProgress`
5. `InProgress` → `Completed`

### Статуси матчів
1. `Waiting` → `CreatingServer`
2. `CreatingServer` → `Ready`
3. `Ready` → `InProgress`
4. `InProgress` → `Completed`

## Помилки та обробка

### Типові помилки
- `400 Bad Request` - некоректні дані
- `404 Not Found` - турнір не знайдено
- `403 Forbidden` - немає прав доступу
- `500 Internal Server Error` - серверна помилка

### Формат помилок
```json
{
  "success": false,
  "message": "Tournament not found",
  "errors": ["Detailed error message"]
}
```

# Аналіз ідеї кіберспортивної платформи для CS2

## Стратегії диференціації (чим виділитися)

Один із ваших ключових запитів: «чим можна вирізнитись, щоб був сенс розробки?». Це слушне питання – заходити на ринок, де вже є великі гравці, без унікальної торгової пропозиції (USP) доволі ризиковано. Якщо зробити просто «ще один Faceit/PlayersLounge», то як переконати гравців кинути набридле, але знайоме і перейти на нове, невідоме? Ось кілька ідей для диференціації:

### Локальний фокус і ком'юніті

Можна позиціонувати проєкт як платформу, створену "геймерами для геймерів" певного регіону чи спільноти. Наприклад, українська кіберспортивна платформа для CS2 з інтерфейсом українською (і російською, якщо на весь СНД), зі своїми локальними турнірами, можливо співпрацею з локальними стрімерами. 

Faceit і подібні – глобальні, часто без душі спільноти. Ви ж можете робити акцент на спілкуванні: додати Discord-сервер для користувачів, проводити івенти (наприклад, щотижневі любительські кубки для новачків, або турнір "Кубок України з 1v1" з невеликим призом). Такий ком'юніті-підхід може залучити ядро аудиторії, яка почуватиметься не просто випадковими користувачами, а членами спільноти. 

Крім того, якщо, скажімо, Faceit більш популярний в Західній Європі, а FASTCUP в Росії, то ви можете спробувати стати платформою №1, наприклад, для України, Польщі, країн Балтії – регіонально ближніх. Це ніша, де можна вирости органічно, а потім вже і далі розширитися.

### Кращий досвід новачків і середньорівневих гравців

На Faceit є проблема – високий рівень токсичності і багато сильних гравців, що лякає новачків. FASTCUP теж, судячи з відгуків, страждає від токсичної аудиторії. Ви можете спробувати диференціюватися як платформа з дружнім ком'юніті і справедливим рейтингом. 

Наприклад, впровадити систему репутації: після матчу гравці залишають відгук чи ставлять оцінку опоненту (чи був він ввічливим, чи дотримувався правил). Тих, хто постійно порушує або ображає, можна карати або відсторонювати. Звісно, це нелегко підтримувати, але якщо вийде створити менш токсичну атмосферу, багато гравців це оцінять. 

Також можна зробити розділення за рівнями: наприклад, лобі тільки для новачків (щоб не боялися, що прийде якийсь "Global Elite" і забере їх гроші). Players' Lounge пишається тим, що їхній матчмейкінг зводить гравців схожого рівня, щоби кожен міг виграти. Ви теж можете зробити акцент на чесному матчмейкінгу – наприклад, мати рейтинг ELO і дозволяти ставити гроші тільки з противниками ± однакового рейтингу. Це унеможливить ситуацію, коли про-гравець буде «фармити» новачків на вашій платформі.

### Інноваційні режими та формати

Можна запропонувати щось, чого наразі не дуже є на інших платформах. Наприклад:

**1v1 режим з ставками на регулярній основі** (хто кращий снайпер або хто переможе в дуелі на певній карті). Faceit концентрується на 5v5, а 1v1 — рідкість. FASTCUP проводить AIM-турніри, але як окремі події. Ви ж можете зробити 1v1 дуелі основою, з таблицею лідерів, щоденними змаганнями.

**Wingman 2v2 на гроші**: CS:GO/CS2 має режим 2v2 (Wingman), який люблять багато друзів-напарників. Турніри 2x2 майже ніде не проводяться регулярно. Це може бути ваша ніша: дуети платять невеликий внесок і грають короткі матчі 2v2.

**Короткі "Fast cup" турніри за вечір**: Наприклад, турнір на 4 команди (тобто півфінал і фінал) — можна провести за пару годин. Вступний внесок невеликий з команди, переможець отримує все. Такі міні-турніри можна запускати часто, і вони залучають тих, хто хоче турнірного досвіду, але не має часу на довгі ліги.

**Режим ставок на статистику**: Це більш експериментально, але наприклад, гравці можуть змагатися не прямо один проти одного, а хто більше наб'є фрагів за N матчів. Скажімо, двоє заходять на публічний deathmatch сервер на 30 хвилин, хто зробив більше вбивств – той переможець. Це як ідея для розмаїття контенту, але реалізація потребує теж відслідковування статистики.

### Технічні "фішки"

Якщо у вас є навички і доступ до AI, ви можете інтегрувати розумні функції, які здивують користувачів. Наприклад:

**Автоматичні хайлайти або аналіз матчу**: Після гри система могла б генерувати короткий звіт – хто був MVP, точність стрільби, можливо, навіть нарізку кращих моментів (якщо записувати демо і проганяти через AI для пошуку кіллів). Це важко, але унікально – жодна платформа цього не дає поки що.

**Чат-бот підтримки**: Використайте ChatGPT/Claude для реалізації живого чату підтримки, який відповідає на часті питання користувачів миттєво (наприклад: "Як мені прив'язати Steam?", "Що робити, якщо опонент не з'явився?" – бот може допомогти, спираючись на вашу базу знань). Швидка і дружня підтримка – великий плюс у сервісу.

**AI-виявлення читерства**: Як згадувалося, якщо зможете впровадити алгоритми, що аналізують гру на ознаки нечесної гри – це був би прорив. Але навіть просто швидке реагування на репорти з допомогою AI (що проглядатиме демо і підказуватиме модератору потенційні моменти читерства) вже пришвидшить модерацію.

## Висновки: об'єктивна оцінка ідеї

Підсумуймо найважливіше чесно і по факту:

### Ідея має попит, але не є унікальною

Грати в CS і отримувати приз – цього хочуть багато гравців, і успіх таких платформ як Faceit (40+ млн відвідин на місяць) чи Players' Lounge ($200 млн виплат) підтверджує, що модель працює. Водночас, ринок конкурентний: вже є сервіси, які пропонують подібне. Вам доведеться стикатися з прямою конкуренцією, особливо з FASTCUP у ніші користувацьких турнірів CS2 та з різними "wager" сайтами. Це не означає, що місця нема – просто треба переграти конкурентів у чомусь.

### Ризики високі

Чесно кажучи, запустити і успішно розкрутити такий проект – завдання не з легких. З точки зору розробки, вам треба вирішити питання інтеграції з грою та анти-чіту, що вже само по собі складно. З точки зору бізнесу – потрібна критична маса гравців і довіра спільноти, що потребує часу і вкладень. 

Також юридичні ризики: можливо доведеться оформляти все як бізнес з усіма витратами, слідкувати за законністю у різних країнах, не допустити участі неповнолітніх тощо. Є ризик, що без належної безпеки можуть бути шахрайства (чи то фінансові, чи то читерські), які нашкодять репутації. Тобто, треба готуватися до проблем і швидко на них реагувати.

### Команда і ресурси обмежені

Ви згадали, що фактично команда – це ви як програміст плюс "команда" з Claude.AI та ChatGPT (тобто, AI помічники). Це кумедно і креативно, але варто розуміти, що живих людей для різних задач теж потрібно. AI не запустить вам маркетингову кампанію і не домовиться зі стрімером 🙂. 

Добре, що ви маєте навички програмування – значну частину роботи зробите самостійно без коштів. Проте, можливо, варто пошукати співзасновника або помічника, який взяв би на себе маркетинг/операційну діяльність, поки ви займаєтесь кодом. Або, якщо бюджет дозволяє, найняти модератора/менеджера навіть на парт-тайм, коли з'являться користувачі, щоб він відповідав на питання, вів соцмережі. 

Проект такого рівня складності рідко витягується повністю одноосібно. Це не значить, що неможливо – просто будьте готові, що ви виконуватимете багато ролей одразу (програміст, адміністратор серверів, служба підтримки, маркетолог, бухгалтер...). Вигорання – реальна загроза, якщо все звалити на себе, тож краще планувати делегування певних функцій.

### План диференціації необхідний

З того, що ми розглянули, у вас має сформуватися розуміння, як виділитися. Якщо наразі сумніваєтесь в доцільності – подумайте: а яка взагалі мета цього проекту? Заробити грошей, реалізувати себе, допомогти спільноті гравців? 

Якщо, скажімо, ви самі фанат CS і вам подобається ідея, можна почати як хобі-проект для душі, а там побачити. Але якщо ціль – прибутковий бізнес, то треба оцінити, чи зможете ви потягнути конкуренцію. Об'єктивно, Faceit/FASTCUP мають більші ресурси і відомий бренд, переманити їхніх юзерів важко. 

Проте нові ніші (1v1, локальні турніри) ще не зайняті на 100%. Можливо, варто спершу зайняти вузьку нішу і стати в ній найкращими, ніж намагатися охопити "всіх гравців CS2" одразу. Тобто сфокусуйтеся.

### Рекомендація щодо розвитку

Якщо вирішите робити – складіть чіткий покроковий план. Наприклад:

1. **Розробити MVP**: реєстрація через Steam, створення 1х1 матчу, депонування коштів, вручну підтвердження результату, базова виплата переможцю.

2. **Запросити невелику групу тестерів** (друзів, знайомих з CS чи з інтернет-спільнот) і провести кілька тестових дуелей/турнірів. Зібрати фідбек: що працює, де проблеми.

3. **Вдосконалити**: додати анти-чіт заходи (напр. вимога запису відео, вбудувати EasyAntiCheat, якщо можливо), зробити гарний UI, виправити баги.

4. **Підготувати маркетинг-матеріали** (промо-сайт, FAQ, Discord сервер, можливо трейлер-відео).

5. **Провести перший публічний турнір/акцію** – це дасть стартовий PR.

6. **Далі** – постійно займатися спільнотою, ітеративно додавати нові функції (скажімо, рейтингова система ELO, підтримка 2v2, і т.п.).

7. **Паралельно** – моніторити конкурентів, вчитися на їхніх помилках (наприклад, слідкувати за відгуками на FASTCUP і Faceit, щоб не повторювати їхні мінуси, а навпаки, вирішувати їх у себе).
